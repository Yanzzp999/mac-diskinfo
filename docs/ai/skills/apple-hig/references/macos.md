# macOS Design Patterns

This is the most relevant reference for `mac-diskinfo`.

## Windows

- Standard windows should be resizable and remember sensible size and position.
- Utility panels should stay lightweight and contextual.
- Use native traffic-light window controls and standard titlebar behavior.

## Navigation And Layout

- Prefer sidebars when there are multiple sections or categories.
- Use split views for master-detail flows.
- Tabs are appropriate for multiple documents or parallel views.

## Toolbars And Menus

- Put primary actions in toolbars.
- Keep menu-bar conventions familiar: File, Edit, View, Window, Help.
- Offer context menus for item-level actions.

## Controls

- Use native-feeling buttons, pop-up buttons, segmented controls, search fields, sliders, and steppers.
- Keep control density moderate and labels clear.

## Keyboard And Pointer

- Design for mouse, trackpad, and keyboard from the start.
- Add standard shortcuts for common actions.
- Maintain predictable tab order and keyboard focus.

## Integration

- Support light and dark appearance.
- Respect reduced motion and increased contrast settings.
- Avoid phone-style bottom tab bars unless there is a strong product reason.

## Practical Bias For This Repo

- Prefer a desktop information layout over stacked mobile cards.
- Use tables, lists, and detail panels for disk information.
- Favor clarity and scanability over ornamental visuals.
